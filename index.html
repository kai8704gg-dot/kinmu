基本の流れ

  1. メンバーを登録する — メンバー欄で名前と役職（師長／副師長／一般職(リーダー)／一般職／新人）を設定します。
  2. 希望や休暇を入れる — 各自がマイページで希望勤務（公休の希望など）を入力します。師長は全員分を編集できます。
  3. ⚡自動作成 — 3つの方式から選ぶと、条件を満たす勤務表が自動で作られます。
  4. チェックと手直し — 条件違反は赤枠と一覧で表示されます。セルをクリックして選び直せます。
  5. 出力 — CSV出力（Excelで開けます）または印刷（A4横）します。

勤務表の操作（師長・管理者）

  • セルをクリックすると勤務の選択メニューが開きます。選ぶだけで入力できます。
  • 名前をクリックすると個人ページが開きます。名前をドラッグすると行を並び替えられます。
  • 右端に個人ごとの回数、下に日ごとの人数が集計されます。条件に合わない箇所は赤く表示されます。

マイページ（全員）

  • 右上の「👤マイページ」から自分のページを開きます。
  • 希望勤務 — 日付をクリックして希望（希望休など）を選びます。自分の希望だけ変更できます。
  • パスワード変更 — 自分のログインパスワードを設定・変更できます。
  • 希望の書き出し — 希望をファイルにしてLINEなどで師長に送れます。師長は「📥希望ファイルを読み込み」で取り込みます。

自動作成の3方式

  • ① 希望を全て反映 — 全員の希望を必ず反映します。不可能なときは作らずに理由を表示し、②③を選べます。
  • ② 可能な限り反映 — 条件を優先しつつ希望をできるだけ反映します。反映できなかった希望は一覧で表示されます。
  • ③ 希望を考慮しない — 条件のみで作成します。

自動作成は「長日勤 → 翌日準夜 → 深夜 → 休み」の4日ブロックで夜勤を回します。このため長日勤も毎日、夜勤と同じ人数になります。

個人ページの設定（師長・管理者）

  • 時短勤務 — 「時短勤務にする」にチェックし、勤務時間を入力します。時短の人は夜勤・長日勤に入りません。
  • 長期休暇 — 産休・育休などの期間を登録すると、勤務表でグレー表示になり割り当てから外れます。公休の必要日数も在籍日数に合わせて自動調整されます。
  • パスワードのリセット — 管理者はこのページから各メンバーのパスワードを再設定できます。

条件（チェックする規則）

⚙設定の「条件チェック」で、各規則のオン・オフと数値を変更できます。オフにした規則は警告の対象外になります。

条件の追加 — 「＋条件を追加」から新しい条件を作れます（✚マーク付きで一覧に並び、オン・オフや削除ができます）。作れる条件の種類:

  • 連続勤務の上限（例: 連続勤務は5日まで）
  • ある勤務の1日あたりの人数（ちょうど／以上／以下）
  • ある勤務の1人あたり月間回数（ちょうど／以上／以下）
  • ある勤務の翌日を制限（例: 深夜の翌日は公休のみ）
  • ある役職を特定の勤務に入れない（例: 新人は深夜に入れない）
  • ある勤務に特定の役職を〇名以上（例: 日勤にリーダー2名以上）

※ 追加した条件は警告チェックに反映されます。自動作成は標準条件で組み立てるため、追加条件の違反が出た場合はセルをクリックして手直ししてください。

  • 準夜・深夜は設定人数ぴったり（新人が入る日は＋1名の5名体制）
  • ゼット勤務（準夜→翌日深夜が同一メンバー）
  • 公休の日数（標準8日）
  • 日勤に師長（不在日は副師長）が必要
  • 長日勤の翌日は準夜か公休
  • 日勤・長日勤・準夜・深夜にリーダー1名以上（師長・副師長もリーダー扱い）
  • 夜勤は1人あたり月5回以内（標準）
  • 新人の夜勤は月1回のみ
  • 長日勤と夜勤の回数は同数
  • 時短勤務者に夜勤・長日勤を入れない

期間の開始日（締め日対応）

⚙設定の「期間の開始日」を変えると、「21日〜翌月20日」のような期間で勤務表を作れます。「1」なら通常の月初〜月末です。

※ 開始日の変更は運用開始前に行うことをおすすめします（既存データの日付対応がずれるため）。

スマートフォンでの利用

  1. このHTMLファイルをLINEやメールでスマホに送って開きます（画面は自動でスマホ用になります）。
  2. スマホでも同じようにログインし、マイページで希望を入力します。
  3. 「📤希望を書き出す」でファイルを保存し、師長に送ります。師長はパソコンで取り込みます。

※ スマホとパソコンのデータは自動では同期されません。受け渡しは希望ファイルで行ってください。

クラウド同期（全端末でデータを共有）

Firebase（Googleのクラウド、無料プラン）を使うと、職場のPC・自宅のPC・各自のスマホが同じデータをリアルタイムで共有できます。スマホで入力した希望が、そのまま師長の画面に反映されます。

準備（管理者が1回だけ行います。同梱の「クラウド同期の設定手順」に画面つきの詳細があります）:

  1. Googleアカウントで console.firebase.google.com を開き、プロジェクトを作成
  2. Authentication →「匿名」ログインを有効化
  3. Firestore Database → データベースを作成（本番モード、東京: asia-northeast1）→ ルールを手順書のとおり貼り替え
  4. プロジェクト設定 → ウェブアプリを追加 → 表示される apiKey / authDomain / projectId をメモ
  5. このアプリに管理者でログイン → ⚙設定 →「☁クラウド同期」に3つの値と病棟ID（推測されにくい好きな文字列）を入力 →「保存して接続」
  6. 他の端末でも同じHTMLを開き、同じ4つの値で接続すると同期されます
  • ヘッダーの「☁同期中」が緑なら共有されています。
  • 初めて接続した端末のデータがクラウドの初期データになります。2台目以降はクラウドのデータで上書きされます。
  • ※ 病棟IDと設定値を知っている人はデータにアクセスできます。IDは長めの文字列にし、部外者に教えないでください。

URLで開けるようにする（おすすめ）: 手順書のSTEP 7のとおり無料のGitHub PagesにこのHTMLを公開すると、各自がURLを開くだけで使えます。さらに管理者が⚙設定の「🔗共有用リンクをコピー」で作ったリンクを配れば、開くだけで接続設定まで完了します（職員は自分のユーザーでログインするだけ）。

データの保存と引き継ぎ

  • データは開いているパソコンのブラウザ内に自動保存されます。ファイル自体には入っていません。
  • 別のパソコンへ移すときやバックアップは、⚙設定の「💾データを書き出し」→「📂データを読み込み」を使います。
  • ブラウザの履歴削除（サイトデータの削除）でデータが消えることがあります。定期的な書き出しをおすすめします。
  • ※ ログインは簡易的なものです。厳密なセキュリティが必要な情報は保存しないでください。

月をまたぐ夜勤について

期間の最終日の長日勤チームは、翌期間の初日に準夜へ入る想定です。翌期間を作成したら、初日の準夜・深夜メンバーを前期間の末尾に合わせて手直ししてください。
